routing은 매핑 or 이동 이라고 생각한다.

express routing : ejs, restapi 서버에서만든 페이지를가져오는
vue routing: 

             (test1.ejs를 줘..)
client   -------------------------> server 
                                    index.ejs
                                    test2.ejs
                                    test1.ejs
                                    등 다양한 페이지를 미리 만들어둔다.
                                    그래서 그거에 맞는거를 client에 보내줌 : 이거는 서버사이드 랜더링
        
            (all.html을 줘..)
         ------------------------->  all.html을 보내줌 
all.html <-----------------------
all.html에서
필요한 페이지 찾아서
화면에 띄운다.  : 이거는 spa(single page application), 클라이언트 사이드 랜더링
페이지 전체를 통으로 갈아버리는게 아니라 컴포넌트로 내용만 바꾸는거
vue router를 이용해서.



클라이언트사이드랜더링의 장점 (=서버사이드랜더링의 단점)
: 페이지전환이 빠르다
: 한번 다운로드되면 페이지 라우팅이 빠르다

클라이언트사이드랜더링의 단점
:초기 다운에 시간이 걸린다.

서버사이드랜더링의 장점
: 다운받을 정보가적다
: 초기랜더링에 시간이 적게 걸린다.
